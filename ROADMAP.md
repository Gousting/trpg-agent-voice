# TRPG Agent Voice 路线图

语音版是"嘉宾模式"（形态C）的基石。核心挑战不是 STT 精度，而是**语音交互的节奏感**——不能让说话的人等 KP 慢慢生成。

## Phase 1：基础语音管道（2 周）

- [ ] **麦克风采集** — PyAudio 多设备支持，音量指示器。
- [ ] **faster-whisper 集成** — 实时 STT，< 1s 延迟，中文优化。
- [ ] **语音活动检测（VAD）** — 不说话时不送 STT，减少噪音输入。
- [ ] **CLI 语音模式** — 终端内 `trpg-voice` 命令启动语音会话。

## Phase 2：语音体验（1 月）

- [ ] **TTS 朗读** — KP 叙述自动转语音（edge-tts 首发，Piper 本地备选）。
- [ ] **多角色音色** — 不同调查员/NPC 用不同 TTS 音色区分。
- [ ] **打断机制** — 玩家说话时 KP 停止 TTS，像真人对话一样自然。
- [ ] **语音指令** — "检定侦查""使用急救"等关键词触发快捷操作，不用等回合。

## Phase 3：直播集成（2 月）

- [ ] **Discord 语音桥接** — Discord bot 加入语音频道，采集多人说话。
- [ ] **说话人分离** — 多人同时说话时区分谁在说什么（diarization）。
- [ ] **微信语音接入** — 微信小程序录音 → 后端 STT → 参与跑团。

## 技术难点

- **STT 延迟 vs 精度**：faster-whisper 的 tiny 模型快但错多，large-v3 准但慢。需要按硬件自适应。
- **多人场景**：Discord 频道 3+ 人同时说话，需要 diarization + 重叠语音处理。
- **TTS 节奏**：KP 旁白通常 2-3 段话，TTS 不能一口气读完——需要按标点分段，等玩家反应。
